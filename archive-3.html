Welcome To My New Site

Posted January 20th, 2012 in Development

So here it is. I've been working on a static site generator for the past couple of weeks and I finally have things put together enough to start running my site off of it. The project is called markd and it's hosted on github.

The overview of how the system works is you create files (posts) that are written in Markdown syntax. markd will then generate a static HTML version of your blog from those files. It makes everything you see here.

I would say it's still very alpha at the moment, and even when it is stable it will still be something of more interest to developers rather than the average blogger. The installation and setup instructions basically don't exist, but will be coming in the future.

Here are the basic reasons why I wrote it:

  • I needed a new hobby project :)
  • Having a static HTML site just seemed to make sense for various reasons
  • It was a bit of an exercise in some development areas that I wanted to work out
  • Creating Markdown files is so easy and the way I have this all set up, makes writing a new post both simple and something I can easily enjoy in my favorite text editor
  • Really, it's just something that I had a strong interest in and decided to run with it. I'm happy with how it's turned out.

If you're curious how it works, here is how I have things set up:

  • I write blog posts in a text editor, with the Markdown syntax. They're saved on Dropbox so I can access them from any computer.
  • I run markd which is a PHP-CLI set of scripts that generates the HTML for my home page, single post pages, archives, etc.
  • Those files are then automatically published to my web server using a set of not-included shell scripting.

I'm enjoying working on it and have plans for things to do in the future like enhancing the theme engine, adding hooks for a plugin system and support for some different post formats. At this point you might be thinking this is starting to sound a lot like WordPress, or many of the other blogging engines, including the others that create static sites. Well, you'd be correct.

I wanted this project to do things the way I wanted them done. It's definitely taking hints from WordPress, and I also intend to take some hints from the Carrington Theme Framework for the theme engine.

I'm looking forward to where this might go. Who knows, maybe working on markd might get me blogging frequently again since I'll be developing it and want to try things out.

My language

Posted May 26th, 2011 in Articles

I have had programming language debates in the past.  I've recently been having similar discussions and realized some things.  Arguing one language against another is a religious debate that will never be won.  I realized it wasn't about #winning.  It would be more valuable to change the discussion to why I chose PHP and continue developing with it to this day.

Total disclosure, one of the key reasons I chose PHP originally was because it was easy to find an inexpensive web host that supported it.  In fact the web host I was using at the time already supported it.  I didn't have to worry about licensing fee's, there were many hosts supporting it, and I was already using web applications built on it.  Those are pretty simple reasons, and certainly reasons that could have changed over time.  For instance, unlike when I was 20 and learning PHP, these days I could afford the licensing fee's if I chose to.  The key thing that has kept me with PHP is the documentation and community.

The documentation for PHP itself is unrivaled with other languages from what I've seen.  PHP.net is a fantastic learning resource and was a huge help when I was getting started.  I could easily browse available functions, and didn't need to search across tons of blogs for likely out of date information.  If I needed to sort an array I could look up sort() and as well as providing me the syntax/parameters, at the bottom of the page it would list other array sorting functions which I could check to see if they better met my needs based on what I was doing in my application.  This matured documentation also exists in other places within PHP.  For example in WordPress, the documentation is fairly strong.  It's not always completely up to date, but the fact it's as comprehensive as it is places it head and shoulders above many other open source programs.

The community is definitely the key feature for me with PHP.  I find it both welcoming and accepting of various skill levels.  I find folks in the PHP community commonly focus on getting stuff done and eagerly teach and share with each other.  There's not much meaningless focus on academic debates or pontificating to boost one's own ego.  That's not to say those things never happens in the PHP community, but I find it happening less often in the world of PHP.

I'm not entirely sure why lots of folks like to consider PHP the red-headed stepchild of web development, but they do.  When developers of other languages do start attacking PHP or my use of it, they're usually just reminding me of why I like my choice of PHP in the first place.  It would probably be a more meaningful discussion for all of us if they'd instead try to show me why they chose their preferred language instead of trying to compare it feature for feature with PHP and simply answering my questions while respectfully understanding if I don't agree about whether I should be using their preferred language.
